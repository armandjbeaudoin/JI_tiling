# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Penrose-like quasicrystal tilings generated by projecting a 5D integer lattice onto a 2D plane (cut-and-project method). Lattice vertex coordinates are interpreted as exponents of prime harmonics {3, 5, 7, 9, 11}, connecting tiling geometry to just intonation (JI) pitch ratios. The tiling is used as a spatial score in SuperCollider.

pip install -r requirements.txt

## Data pipeline

1. Run `notebooks/penrose5D.ipynb` end-to-end ‚Äî this is the only step that produces data.
2. It writes three CSV files to `Data/`:
   - `verts_5Dprojection.txt` ‚Äî 2D coordinates of each tile's four corners
   - `indices_5Dprojection.txt` ‚Äî 5D lattice indices for each tile vertex
   - `faces_5Dprojection.txt` ‚Äî face-direction index (determines tile shape/color)
3. Open `tiling.scd` in SuperCollider and evaluate blocks in order.

## Architecture

### Python layer

- **`src/ji_tiling.py`** ‚Äî shared utilities imported by all notebooks via `sys.path.insert(0, '../src')`.
- **`src/coxeter_a4.py`** ‚Äî shared SageMath utilities for A4 Coxeter theory, imported similarly.
- **`notebooks/`** ‚Äî Jupyter notebooks that must be run from the `notebooks/` directory or via JupyterLab at repo root. Paths to `Data/` use `'../Data/'`.
- when modifying notebooks, create a new cell instead of modifying, unless explicitly directed.  Once code is debugged, cells that are replaced by new code will be deleted manually.
Key functions in `ji_tiling.py`:
- `basis2D(eigenVectors, k, l)` ‚Äî builds the (2√ó5) projection matrix from eigenvector pair k, l of the 5D cyclic permutation matrix
- `in_hull(p, hull)` ‚Äî convex-hull membership test (wraps `scipy.spatial.Delaunay`)
- `polygonDataFunction(point, dir_)` ‚Äî returns the four 5D corners of a rhombus tile
- `subSet`, `unitVector`, `nMemberQ` ‚Äî combinatorial/geometric helpers

### SuperCollider layer (`tiling.scd`)

The file is organized into named blocks separated by `//===` dividers, meant to be evaluated sequentially in the SuperCollider IDE (Cmd+Return per block).

**Key SuperCollider globals:**
- `~baseFreq` ‚Äî reference frequency in Hz (default 261.63)
- `~intervals` ‚Äî current active interval array (index into `~latticeIntervals`)
- `~constantTone` ‚Äî if true, uses persistent `~toneArray` synths updated via `.set`; if false, spawns new synths per click
- `~ampRef` ‚Äî amplitude reference scalar for click-position-based mixing

### Path conventions

| Context | Path style |
|---|---|
| `src/ji_tiling.py` / `src/coxeter_a4.py` import in notebooks | `sys.path.insert(0, '../src')` |
| `Data/` access in notebooks | `'../Data/'` |
| `Data/` access in SuperCollider | `thisProcess.nowExecutingPath.dirname +/+ "Data/"` |
